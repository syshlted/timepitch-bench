# dsp-bench session — 2026-05-20

Working transcript of the session that designed the Shepard tone test and the
spectral-envelope-preservation metric. Captured from memory after the session;
prose paraphrased, numerical results verbatim from the benchmark runs.

## Starting point

`dsp-bench/` already had a working harness comparing Signalsmith Stretch,
SoundTouch, and Rubber Band on sine, sweep, impulse, and white-noise signals,
with CPU / peak-RSS / pitch-error (cents) reporting and an optional
algorithmic-latency mode. See [../README.md](../README.md).

## 1. Baseline battery (the four existing signals)

Ran seven configurations across all three stretchers to establish current
behavior on desktop Linux x86_64 (5-second buffers, 48 kHz stereo).

**Realtime factor (higher = faster than realtime):**

| Library | Sine 1.5x | Pitch +1oct | Pitch -1oct | Sweep 1.5x | Noise pitch (1024) | Impulse 1.5x | Neutral 1.0x |
|---|---|---|---|---|---|---|---|
| signalsmith | 57x | 59x | 63x | 55x | 54x | 57x | 65x |
| soundtouch | **513x** | **247x** | **245x** | **505x** | **242x** | **567x** | **433x** |
| rubberband | 10x | 9x | 10x | 10x | 8x | 12x | 11x |

**Pitch accuracy (sine, cents):** all three within ±0.5 cents — quality on
pure tones is essentially equivalent.

**Peak memory:** signalsmith ~10–13 MB, soundtouch ~11–14 MB, rubberband
~17–19 MB.

**Algorithmic latency:** rubberband 2048 frames (~43 ms), signalsmith 2880
(~60 ms), soundtouch 3660 (~76 ms).

Observation: synthetic sine is a weak discriminator. We need a signal that
exercises the spectrum more.

## 2. Designing the Shepard tone

Proposed as a stress test: ten octave-spaced partials sweeping continuously
upward at 0.5 oct/sec under a fixed Gaussian envelope in log-frequency
(center = log2(500 Hz), sigma = 2 octaves). For stretchers this means:

- **Time-stretch**: instantaneous partial frequencies at any equivalent input
  position should be preserved; only the rate of sweep changes.
- **Pitch-shift**: every partial multiplied by the same ratio; octave
  structure preserved.
- **FFT check**: averaged spectrum should show peaks at octave spacing.

Implementation:

- Added `SignalKind::Shepard` to [../src/signals.h](../src/signals.h) /
  [../src/signals.cpp](../src/signals.cpp). Closed-form phase for the
  exponential sweep: `φ(t) = 2π f₀ (2^(rate·t) − 1) / (rate·ln 2)`.
- Added `find_top_peaks()` in [../src/fft.h](../src/fft.h) /
  [../src/fft.cpp](../src/fft.cpp) — parabolic-interpolated peak picker
  ranked by magnitude.
- Extended `QualityReport` with Shepard fields: input/output peak lists,
  median adjacent-octave ratio, observed pitch ratio (median of
  nearest-input-peak ratios in log-freq).
- Quality scoring runs on mid-buffer windows of input and output (8192
  samples), which sits past stretcher latency tails.

## 3. First Shepard run

Octave preservation was excellent across the board (median ratio
1.999–2.001 in every test). Pitch ratio detection mostly worked, with two
surprises:

**Pitch +1 octave looked "wrong" for the right reason.** Output partials
coincide exactly with the input partials one octave above — so the
nearest-input-peak matcher snapped to a ratio of ~1, not 2. That's the
Shepard illusion working as designed: a pitch shift of an exact octave is
spectrally indistinguishable from identity on this signal. Use non-octave
shifts to actually probe pitch accuracy.

**Signalsmith showed a 69-cent error at pitch 1.3348 — but was it real?**
Cross-checked with the existing sine pitch-shift test at the same ratio:

| Library | error (cents) on sine, 1.3348 |
|---|---|
| signalsmith | 1.02 |
| soundtouch | 0.17 |
| rubberband | 0.16 |

So signalsmith is sub-cent accurate on raw pitch. The 69 cents was an
artifact of comparing **mid-windows of a *sweeping* signal** across
stretchers with different latencies. Signalsmith's 60 ms latency means its
mid-output window corresponds to a slightly earlier point in the input
sweep than the other libraries'. With sweep_rate = 0.5 oct/sec, even small
time misalignments turn into tens of cents.

## 4. Stationary Shepard mode

Added `--shepard-sweep-rate <oct/sec>` (default 0.5). Passing 0 produces
stationary partials, eliminating the sweep-vs-latency interaction. With
stationary partials:

| | identity | pitch 1.3348 | pitch 2.0 | time 1.5x |
|---|---|---|---|---|
| signalsmith | 1.0000 | 1.3364 (+2¢) | 1.0005 | 1.0000 |
| soundtouch | 1.0000 | 1.3348 (perfect) | 1.0001 | 1.0000 |
| rubberband | 1.0000 | 1.3348 (perfect) | 1.0000 | 1.0000 |

Confirmed: the earlier 69-cent error was 100% sweep-vs-latency, not a
stretcher quality issue.

## 5. Spectral envelope-preservation metric

Insight: the Shepard tone's amplitude envelope is Gaussian in log-frequency
by construction. After pitch shift R, the envelope is theoretically
shifted to center = `log2(500·R)` with sigma unchanged at 2 octaves.
Fitting `log(magnitude) = A + B·log2(f) + C·log2(f)²` to the detected
peaks recovers `μ = -B/(2C)` and `σ = √(-1/(2C))` — a clean characterization
of how each stretcher preserves spectral balance.

Implementation: `fit_quadratic()` (3×3 normal equations with partial
pivoting) plus a thin wrapper `fit_gaussian_envelope()` over the peak
list. Stored on `QualityReport`. Applied to both input and output peaks.

First results were promising but the absolute centers were biased ~1 Hz
low at identity (analysis-side: FFT-window amplitude bias on edge peaks
affects the fit). Sigma was biased ~0.05 oct low for the same reason.
The bias was consistent across all three stretchers, suggesting it would
cancel in a relative measurement.

## 6. Input-relative comparison

Idea: fit the same Gaussian to the *input* peaks too. Report
`center_error_cents = (observed_shift_oct − expected_shift_oct)·1200` and
`sigma_error = output_sigma − input_sigma`. By comparing output to input
rather than to theory, the analysis-side bias cancels.

Result: identity errors collapsed to sub-cent across all three libraries
(was 1–1.5 Hz on absolute, became 0.2–1.4 cents on input-relative).
Clean separation of stretcher-side spectral coloration emerged. Findings
written up in [shepard_findings.md](shepard_findings.md).
